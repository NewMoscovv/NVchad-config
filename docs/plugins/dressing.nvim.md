# `dressing.nvim`

## Что это

`dressing.nvim` улучшает стандартные интерфейсы `vim.ui.select()` и `vim.ui.input()` в Neovim.

Проще говоря, это плагин, который делает встроенные окна выбора и ввода аккуратнее и удобнее.

В этой конфигурации он используется как базовый UI-слой для:

- выбора элементов из списка
- ввода строковых значений
- интеграции с плагинами, которые используют `vim.ui.select()` и `vim.ui.input()`

## Где подключён

[lua/plugins/init.lua](/Users/kirsanov.mihail5/.config/nvim/lua/plugins/init.lua)

```lua
{
  "stevearc/dressing.nvim",
  lazy = false,
  opts = {},
}
```

## Как загружается

- `lazy = false`

Это значит:

- плагин загружается сразу при старте Neovim
- он доступен до того, как другие плагины начнут вызывать `vim.ui.*`
- отдельного события (`event`, `ft`, `cmd`) для него не задано

## Что делает в этой конфигурации

Сейчас плагин подключён с дефолтными настройками:

```lua
opts = {}
```

То есть кастомной настройки поверх стандартного поведения пока нет.

Практический эффект такой:

- окна выбора становятся аккуратнее стандартных
- окна ввода выглядят лучше базового `input()`
- если какой-то плагин просит выбрать действие, файл, код-экшен или ввести значение, это часто проходит через `dressing.nvim`

## Как это выглядит в работе

Отдельных команд и хоткеев для `dressing.nvim` в конфигурации не добавлено.

Ты используешь его не напрямую, а косвенно через другие функции Neovim и плагины.

Типичные примеры:

- переименование символа через LSP
- выбор действия из списка
- подтверждение или ввод значения в floating-окне

## Как проверить, что он работает

Самый простой способ:

1. Открыть любой проект с LSP
2. Вызвать действие, которое требует выбор или ввод
3. Убедиться, что появляется аккуратное floating-окно, а не сырой встроенный prompt

Также можно проверить через Lua:

```vim
:lua print(vim.inspect(vim.ui))
```

Или посмотреть, что плагин загружен:

```vim
:Lazy
```

В списке должен быть `dressing.nvim`.

## Ограничения

- В этой конфигурации у плагина нет отдельных маппингов
- Сейчас он используется на дефолтных настройках, без кастомного оформления
- Это не picker уровня `Telescope`, а именно улучшение стандартного `vim.ui.*`

## Важный статус

Репозиторий `dressing.nvim` архивирован автором 12 февраля 2025 года и переведён в read-only.

Это не означает, что плагин сломан. Это означает, что:

- он всё ещё может нормально работать
- активного развития больше нет
- в будущем можно рассмотреть замену, если захочется обновить UI-слой

Автор рекомендует смотреть в сторону `snacks.nvim` для `vim.ui.input`, а для `vim.ui.select` часто хватает возможностей уже установленного picker-плагина.
