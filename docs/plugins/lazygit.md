# `lazygit`

## Что это

`lazygit` — это отдельный TUI-клиент для Git.

Он не заменяет встроенные команды Neovim и не является Lua-плагином, но даёт быстрый интерфейс для повседневной работы с:

- staged / unstaged изменениями
- коммитами
- ветками
- stash
- rebase / cherry-pick / revert
- просмотром diff

В этой конфигурации `lazygit` открывается прямо из Neovim во floating terminal.

## Как установлен

На этой машине `lazygit` установлен через Homebrew:

```bash
brew install lazygit
```

Проверка:

```bash
lazygit --version
```

## Как подключён в этой конфигурации

Подключение сделано через пользовательскую команду и маппинг в [lua/mappings.lua](/Users/kirsanov.mihail5/.config/nvim/lua/mappings.lua):

- команда `:LazyGit`
- хоткей `<leader>gg`

Что делает интеграция:

1. Определяет корень Git-репозитория по текущему буферу
2. Открывает `lazygit` в floating terminal с рамкой
3. Запускает его в корне найденного репозитория
4. После выхода из `lazygit` автоматически закрывает окно

Если текущий файл не принадлежит Git-репозиторию, Neovim покажет предупреждение.

## Как открыть

### Из Neovim

```vim
:LazyGit
```

или:

```vim
<leader>gg
```

### Из терминала

```bash
lazygit
```

Откроется репозиторий из текущей директории.

### Открыть конкретный репозиторий

```bash
lazygit -p /path/to/repo
```

## Базовая навигация

Самое важное, что стоит запомнить с первого дня:

| Клавиша | Что делает |
|---------|------------|
| `j` / `k` | Вниз / вверх по списку |
| `h` / `l` | Перейти между панелями |
| `<tab>` | Переключить панель |
| `1..5` | Прыжок в конкретную панель |
| `0` | Фокус на главную панель |
| `<enter>` | Открыть / подтвердить |
| `<esc>` | Назад |
| `q` | Выйти из `lazygit` |
| `?` | Показать контекстную помощь / доступные действия |
| `/` | Поиск |
| `n` / `N` | Следующее / предыдущее совпадение |
| `R` | Обновить данные |

Если забыл клавиши в конкретной панели, жми `?`. Это самый полезный хоткей в начале.

## Основные панели

Обычно в `lazygit` ты работаешь в нескольких зонах:

- `Files` — изменения в файлах, staging, discard, stash
- `Branches` — переключение веток, rebase, merge, upstream
- `Commits` — история коммитов, cherry-pick, revert, interactive rebase
- `Stash` — сохранённые stash
- `Main / Diff` — подробный просмотр изменений

## Самый частый workflow

### 1. Посмотреть изменения

Открой `lazygit` и перейди в панель `Files`.

Обычно дальше сценарий такой:

- `j` / `k` — выбрать файл или hunk
- `<space>` — stage / unstage выбранного изменения
- `a` — stage / unstage всё
- `<enter>` — раскрыть файл или перейти глубже

### 2. Сделать коммит

В панели `Files`:

- `c` — коммит
- `C` — коммит с открытием редактора сообщения
- `A` — amend последнего коммита
- `w` — коммит без hook

Типовой сценарий:

1. Выделяешь изменения через `<space>`
2. Нажимаешь `c`
3. Вводишь сообщение коммита
4. Подтверждаешь `<enter>`

### 3. Выкинуть изменения

В панели `Files`:

- `x` — discard выбранного изменения
- `D` — reset options / дополнительные варианты сброса

С этим надо быть аккуратным: discard может быть необратим.

### 4. Работать со stash

В панели `Files`:

- `s` — stash all changes
- `S` — открыть меню stash-операций

В панели `Stash`:

- `g` — pop stash
- `r` — rename stash

### 5. Работать с ветками

В панели `Branches`:

- `c` — checkout выбранной ветки
- `-` — вернуться на предыдущую ветку
- `r` — rebase текущей ветки на выбранную
- `R` — переименовать ветку
- `M` — merge выбранной ветки в текущую
- `u` — настроить upstream
- `f` — fetch
- `T` — создать tag

### 6. Работать с коммитами

В панели `Commits`:

- `s` — squash вниз
- `r` — rename commit
- `A` — amend to commit
- `t` — revert commit
- `C` — скопировать commit для cherry-pick
- `V` — вставить commit
- `B` — пометить commit как базовый для rebase
- `i` — начать interactive rebase
- `y` — скопировать атрибут коммита

## Что удобно делать через Lazygit, а что нет

### Удобно

- быстро смотреть diff
- stage / unstage по файлам и ханкам
- делать обычные коммиты
- переключать ветки
- stash / pop stash
- rebase и cherry-pick без постоянного ручного ввода команд

### Лучше оставить shell

- очень сложные одноразовые Git-команды
- массовые scripted-операции
- тонкая работа с редкими флагами `git`
- ситуации, где ты уже точно знаешь нужную CLI-команду быстрее, чем UI

## Полезные команды и настройка

Посмотреть дефолтный конфиг:

```bash
lazygit -c
```

Узнать директорию пользовательского конфига:

```bash
lazygit -cd
```

Обычно конфиг лежит здесь:

```text
~/.config/lazygit/config.yml
```

Туда можно добавлять:

- свои keybindings
- `customCommands`
- темы
- поведение GUI
- интеграции с внешними diff/tooling-командами
