# 📖 Справочник горячих клавиш

> ⚠️ **Важно:** Этот справочник разделён на две части:
> 1. **Реальные хоткеи этой конфигурации** — точно работают, проверены в коде
> 2. **Стандартные хоткеи NvChad** — могут работать, зависят от версии NvChad

---

## 🎹 Лидер-клавиша

**`<leader>` = `Пробел` (Space)**

Все комбинации с `<leader>` начинаются с нажатия пробела.

**Пример:** `<leader>h` = нажать `Пробел`, затем `h`

---

## ✅ Реальные хоткеи этой конфигурации

Эти хоткеи **точно работают** — они определены в файлах конфигурации.

> ℹ️ Для `render-markdown.nvim` в этой конфигурации отдельные хоткеи не добавлены. Управление идёт командами `:RenderMarkdown enable`, `:RenderMarkdown disable`, `:RenderMarkdown toggle`.

### Основные команды

| Клавиши | Режим | Файл | Описание |
|---------|-------|------|----------|
| `;` | Normal | `mappings.lua` | Вход в командный режим (`:`) |
| `fd` | Insert | `mappings.lua` | Выход из режима вставки (`<ESC>`) |
| `<leader>h` | Normal | `mappings.lua` | Открыть главный экран (Dashboard) |

### Git команды

| Клавиши | Режим | Файл | Описание |
|---------|-------|------|----------|
| `<leader>gt` | Normal | `mappings.lua` | Показать изменения в Git через `snacks.nvim` picker |
| `<leader>cm` | Normal | `mappings.lua` | История коммитов через `snacks.nvim` picker |
| `<leader>gg` | Normal | `mappings.lua` | Открыть `lazygit` во floating terminal |

> ⚠️ **Важно:** Git команды работают **только внутри Git-репозитория**. Если файл не в репозитории, покажется предупреждение. `lazygit` тоже открывается только когда для текущего буфера удалось определить корень репозитория.

### Go разработка (LSP)

| Клавиши | Режим | Файл | Описание |
|---------|-------|------|----------|
| `<leader>gi` | Normal (Go) | `configs/lsp.lua` | Организовать импорты Go через `gopls` |

> 💡 **Автоматически:** Импорты организуются **при каждом сохранении** файла `*.go` (автокоманда `BufWritePre`).

### Отладка (DAP)

| Клавиши | Режим | Файл | Описание |
|---------|-------|------|----------|
| `<leader>db` | Normal | `configs/dap.lua` | Поставить/снять точку останова 🔴 |
| `<leader>dc` | Normal | `configs/dap.lua` | Запустить/продолжить отладку ▶️ |
| `<leader>dr` | Normal | `configs/dap.lua` | Открыть консоль отладки (REPL) |
| `<leader>du` | Normal | `configs/dap.lua` | Показать/скрыть панель отладки (dapui) |
| `<leader>dt` | Normal (Go) | `configs/dap.lua` | Отладить ближайший тест |
| `<leader>dl` | Normal (Go) | `configs/dap.lua` | Повторить последнюю отладку |

## 🔧 Стандартные хоткеи NvChad v2.5

> ⚠️ **Внимание:** Эти хоткеи предоставляются **стандартным NvChad** (через `require "nvchad.mappings"` в `mappings.lua`). Они должны работать, но могут измениться при обновлении NvChad.

### Навигация по окнам

| Клавиши | Режим | Описание |
|---------|-------|----------|
| `<C-h>` | Normal | Перейти в окно слева |
| `<C-j>` | Normal | Перейти в окно снизу |
| `<C-k>` | Normal | Перейти в окно сверху |
| `<C-l>` | Normal | Перейти в окно справа |
| `<C-\>` | Normal | Переключиться на предыдущее окно |

**Обозначение:** `<C-x>` = `Ctrl + x`

### Поиск и навигация (Telescope)

| Клавиши | Режим | Описание |
|---------|-------|----------|
| `<leader>fb` | Normal | Поиск по буферам |
| `<leader>fw` | Normal | Поиск по содержимому файлов (live grep) |
| `<leader>ff` | Normal | Поиск файлов по имени |
| `<leader>fh` | Normal | Поиск по help tags |
| `<leader>fo` | Normal | Поиск по недавно открытым файлам |
| `<leader>fz` | Normal | Поиск в текущем буфере |
| `<leader>fa` | Normal | Поиск по всем файлам, включая hidden |
| `<leader>pt` | Normal | Список терминалов |
| `<leader>th` | Normal | Выбор темы NvChad |

### Дерево файлов (NvimTree)

| Клавиши | Режим | Описание |
|---------|-------|----------|
| `<leader>e` | Normal | Открыть/закрыть дерево файлов |

### LSP функции (стандартные)

Эти хоткеи работают **если подключён LSP** (например, gopls для Go):

| Клавиши | Режим | Описание |
|---------|-------|----------|
| `gd` | Normal | Перейти к определению (go to definition) |
| `gD` | Normal | Перейти к объявлению (go to declaration) |
| `gr` | Normal | Найти все ссылки (find references) |
| `K` | Normal | Показать документацию (hover) |
| `<leader>D` | Normal | Перейти к type definition |
| `<leader>ra` | Normal | Переименовать символ через NvRenamer |

> 💡 **Примечание:** В этой конфигурации есть отдельный hotkey `<leader>gi` для **организации импортов Go**.

## 🔍 Как проверить какие хоткеи работают

### 1. Посмотреть все маппинги
```vim
:map
```

### 2. Найти маппинг по клавише
```vim
:map <leader>g
```

### 3. Проверить конкретный маппинг
```vim
:verbose map <leader>gt
```
Покажет где определён маппинг.

Для `lazygit`:
```vim
:verbose map <leader>gg
:LazyGit
```

### 4. Посмотреть маппинги для буфера
```vim
:map <buffer>
```

## 📝 Заметки

### Отключённые хоткеи

В `mappings.lua` закомментирована строка:
```lua
-- map({ "n", "i", "v" }, "<C-s>", "<cmd> w <cr>")
```

Это означает что `<C-s>` для сохранения **не активен** по умолчанию.

### Конфликтующие хоткеи

В текущей документации явных конфликтов больше не отмечается: пользовательские hotkey вынесены отдельно, а стандартные NvChad перечислены в своём разделе.
